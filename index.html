HTML

  • The head element contains information about the webpage.
  • The head element contains the metadata
  • Metadata attributes include description, keywords, anchor, author, viewport, http-equiv
  • Head element contains the Title tag (needed for SEO)
  • Semantic HTML elements - name of element describes the content and role eg head, body (not div)
  • Body tag contains header, main and footer tags
  • Use section tag to separate content in body
  • h1 to h6 tags for headings. h1 is largest, use only once per page.
  • alt tag used for text description in images. Helps screen readers and SEO.

CSS

  • Colours, asthetics, visual layout, organisation
  • Inline - CSS added directly to tag in html. Only impacts that tag.
  • Internal - style added to head of page - impacts all that page.
  • External - CSS in external file. Added to page vie link tag. Same stylesheet be used throughout web site if required.
  • Separation of Concerns - design is modular, easier to maintain, upgrade, reuse (but may impact performance due to extra code to interface with modules)
  • Selector - choose the tag, id, class to apply style to
  • Declaration - style
  • Block elements - take up width of screen. Inline elements - take up width of text only
  • CSS Box - padding, border, margin, content
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Padding and margin can add to the overall size of the element. To stop this use box-sizing
  • Padding margin values can be assigned clockwise or top/bottom and right/left or using 3 options
  • box-shadow attribute can be used to give a shadow effect around an element. Assign values for X axis, Y axis, blur-radius, spread-radius, and colour

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all changes in current working branch to the staging area
  • git commit - m "message" : captures snapshot of currently staged changes and commits
  • git pull origin main: pull the latest version of main into the local branch (origin means sourse is GitHub, main means main branch)
  • git push origin feature/add-git: push local changes to remote GitHub server
  • In GitHub repository - go to branch eg feature/add-js
  • Go to Pull requests tab - click Create pull request
  • Complete PR form - title, description
  • Click Merge Pull Request and Confirm Merge
  • Close issue
  • Make sure changes are in local main branch - in terminal: git checkout main, git pull origin main
  • Deployment - navigate to Prework Study Guide HitHub repo
  • Select Settings
  • Select pages
  • Under source - Branch : main and /root then Save
  • Publishing may take a few minutes - website https://username.github.io/repository-name eg https://helenelee.github.io/prework-study-guide

Javascript

  • Invented by Brendan Eich.
  • Data Types - weak/dynamic typing
  • Prmitive data types - String, Number, BigInt, Boolean, Null, Undefined, Symbol
  • Non-Primitive data type - object
  • Arrays - can store data of different types.
  • falsy - false, 0, -0, 0n, on, "", '', ``, null, undefined, NaN, document.all
  • truthy - everything else
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • "If" statements are used to control flow in javascript
  • "Functions" group code together so it can be called when needed.
  • DRY - Do not Repart yourself - reuse code, do not write twice
